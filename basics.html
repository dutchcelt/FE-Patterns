Panels, boxes, widgets highlighted sections, basically anything that could contain text and a background of some sort. Some sites donâ€™t have them some do and web applications nearly alway have them. They also appear across a number of contexts so defining a pattern is of great value.

Standard box

Each 'box' consists of both margin and padding. In other css markup systems we see the use of 'pod' like in SMACSS.

HTML

<section class="box">
    … general markup …
</section>

LESS

margin: 0;
padding: @interline @block @em;
.box-sizing(border-box);

Box inset

If you need to override the padding of the outer box you can use the 'box‑inset' class. This is useful because you don't need to know what the padding is as it is set by a variable in '.box'.

HTML

<section class="box">
    <header class="box-inset">
        … header text …
    </header>
    <p>
        … paragraph text …
    </p>
</section>

LESS

margin: -(@interline) -(@block) -(@em);
padding: @interline/2 @block;
.box-sizing(border-box);

The weakest link when it come to the web is layout. Help is on the way but it not here yet. For those building enterprise based web applications then it will be a much longer wait. Sometimes a predefined grid can work but usually it's more trouble then it's worth. So defining a pattern that can map to a grid can be way more powerful.

We can think of grids in a number of ways. Either in a sequence of columns or in proportional sections like halves, thirds, quarters and so on. You can map one to the other so think about this when implementing a layout in terms of a grid.

The initial use case for preprocessors is for repeating colours and the need to be able to change these after implementing them. The ability to have gradations based on a single value is a huge time saver. Light, dark, warning, highlight and prime colours are colours you nearly always need.
