Forms

You can't live with them but know you can style them!

The most important mark up to implement consistently are forms. Allow the CSS to take control and keep the HTML simple and the same throughout your all your projects.

Think carefully about the buttons you will need. If can use only the button tag <button> do so, they will give you the most control. Remember that IE6 and 7 have a bug with buttons tags in that they send all the value.

Beware of HTML5 input types, browsers are still inconsistent. The date input type doesn't work very well and has been omitted from these patterns. The examples here have been tested and little or no problems.

Note: Client-side validation has not yet been implemented. Looking into using the HTML5 pattern attribute

Text field

<div> <label for="txt01">Name *</label> <input type="text" id="txt01" required /> </div>

Radio or checkbox entry

<div> <label>Do you wish to receive our newletter?</label> <div> <input type="radio" name="radio" id="r01" /> <label for="r01">Yes</label> <input type="radio" name="radio" id="r02" /> <label for="r02">No</label> </div> </div>

Datepicker

<div> <label for="d01">Date</label> <input type="text" id="d01" class="date"/> </div>

Buttons (right aligned)

<div class="buttons"> <button type="button" class="btn seperate" value="Print"><i class="icon-print"></i>&nbsp;Print</button> <button type="button" class="btn" value="Delete"><i class="icon-trash"></i>&nbsp;Delete</button> <button type="button" class="btn" value="Cancel"><i class="icon-ban-circle"></i>&nbsp;Cancel</button> <button type="submit" class="btn default" value="Save"><i class="icon-save"></i>&nbsp;Save</button> </div>
