Living in a box

It's where your content lives!

Boxes, panels and blocks al have something in common. Usually you want to treat white space in a consistant manner. So your CSS should be able to share share chunks of code without the need to overwrite inherited code.
With LESS this is very important so that you can avoid properties repeating themselves.

To achieve below we needed to make sure block elements with negative margins (heading and footer) are offset by their sibling block elements (sections) to avoid duplication. So that in this case the three headings take up the same amount of space.

  • Simple

    <body> <section class="panel"> … general markup … </section> </body>
    Less code:
    .panel { .box; .rounded-corners(); background-color: @light-color; .box-shadow(); } .panel > footer, .panel > header { .box-inset; } .panel :first-child { .rounded-corners-top(); } .panel :last-child { .rounded-corners-bottom(); } .panel > header ~ section { padding: @interline 0; }

  • Header & footer*

    <body> <section class="panel widget"> <header> <h3>Box with header and footer*</h3> </header> <section>lorem…</section> <footer>* Optional text</footer> </section> </body>
    Less code:

    The '.widget' class extends '.panel'.
    In this case '.widget' only overrides the colour.

    .widget { .text-shadow(@l,@light-color); background-color: @grey1; } .widget header { .gradient(@light-color,@grey2); } .widget footer { background-color: @grey2; }
    * Optional text

  • Highlighted

    <body> <section class="panel widget highlighted"> <header> <h3>Highlighted box</h3> </header> <section>lorem…</section> </section> </body>
    Less code:
    .highlighted { background-color: @yellow1; .gradient(@yellow1,@yellow2); } .highlighted header { color: @light-color; .text-shadow(0 - @l,@dark-color); .gradient(@grey5,@grey6); }
