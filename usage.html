Getting started

LESS is all you need

Building something with the Web Stack you do not need much. HTML, CSS and, maybe, JavaScript. The common HTML patterns we use can be found in 'Content', 'Forms'. 'Tables' & 'Layout' and please feel free to use these in your projects. The 'LESS' section is the hero of our UI patterns and accelerates front-end production considerably.

There are number of files you should use and some that you can omit all together. In fact you don't need most of the files. JavaScript, images, css and fonts you can just skip because these tend to be project or even company specific.

LESS files you really need:

  1. lib.less (contains all variables and mixins)
  2. base.less (reset and realign HTML tags across browsers)
  3. forms.less (basic form settings like label alignment)
  4. main.less (loads all the LESS files in to single CSS file)

The Lib file

The file might look a little excessive. Feel free to pare it down too your own needs. Remember the lib file should not parse out any actual css code and only set variables and mixins for use in the other files.

The base file

This file (re)sets the standard HTML patterns for most browsers. Any additional patterns should be done in your project files as these will tend to specific to it's identity.

One exception to this could be the header tags (h1, h2, h3 etc.) These have set sizes and may prove to be more effective to change them here rather than overriding them elsewhere.

The form file

The form file will render all forms and input fields consistently across browsers. The lib file contains a form section that will allow you to change the position of the label with out altering anything in this file. It's best to leave this file untouched.

The main file

This is the file where you'll include all the files you need. Your template will either call this file to have it compiled by the server or the pre-compiled CSS file (styles/vanilla/css/main.css). Either way all relative image paths should be the same "../images/".

JavaScript

Considering the fact that jQuery is so popular and useful I've added it to this project as an example. Along with other jQuery plugins that have proven there worth; jQuery UI, Validator and DataTables. Last but not least I've added Modernizr to enable HTML5 goodness for those who are unfortunate to have an old browser.
