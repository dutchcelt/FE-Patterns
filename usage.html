Getting started

LESS is all you need

Building something with the Web Stack you do not need much. HTML, CSS and, maybe, JavaScript. The common HTML patterns we use can be found in 'Content', 'Forms'. 'Tables' & 'Layout' and please feel free to use these in your projects. The 'LESS' section is the hero of our UI patterns and accelerates front-end production considerably.

There are number of files you should use and some that you can omit all together. In fact you don't need most of the files. JavaScript, images, css and fonts you can just skip because these tend to be project or even company specific. To this end there is a skeleton setup for you to bootstrap new projects.

LESS files you really need:

  1. FEP-*.less (A collection of library files that contain all variables and mixins)
  2. FEP-base.less (A patterned reset to help HTML tags behave a little more predictable across browsers)
  3. FEP-forms.less (basic form settings like label alignment)
  4. main.less (loads all the LESS files in to single CSS file)

The Lib files

The number of files may look a little excessive. Feel free to pare them down too your own needs. You may not need the icon-fonts lib file for example. Remember these lib files would preferably not parse out any actual css code and only set variables and mixins for use in the other files.

The base file

This file (re)sets the standard HTML patterns for most browsers. Any additional patterns should be done in your project files as these will tend to specific to it's identity.

One exception to this could be the header tags (h1, h2, h3 etc.) These have set sizes and may prove to be more effective to change them here rather than overriding them elsewhere.

The form file

The form file will render all forms and input fields consistently across browsers. The lib file contains a form section that will allow you to change the position of the label with out altering anything in this file. It's best to leave this file untouched.

The main file

This is the file where you'll include all the files you need. Your template will either call this file to have it compiled by the server or the pre-compiled CSS file (styles/legacy/css/main.css). Either way all relative image paths should be the same "../images/".

How to compile LESS into CSS

After you modify the LESS files, you need to run a LESS processor on the main.less file to generate the CSS. It is only necessary to run LESS on main.less since this file automatically imports the other files in the right order.

There is a large choice of LESS processors. If you like the command line, you can use the one that comes with LESS itself. You will need to have Node.js installed to use the CLI tool.

If you want a GUI and use a Mac, you can try LESS.app (which does not require additional downloads).

JavaScript

Considering the fact that jQuery is so popular and useful I've added it to this project. Along with some jQuery plugins to utilise common usage patterns; Keep in View and DataTables. html5shiv is still being added to enable HTML5 goodness for those who are unfortunate to be stuck using IE8. For more on compatibility see below.

The dependancy loader is Requirejs. Fast and asynchronous script loading for any website, large or small.

Compatibility

These patterns are no longer being tested in IE7 or lower as the pattners now make use of more advanced features.

The current requirements are:

  • Internet Explorer 9+
  • Chrome (latest)
  • Firefox (latest)
  • Safari (latest)
