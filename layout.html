Which side is your content on?

Holding it all together!

Here you can see that we're avoiding the cascade to create the layout. In most cases this is a bad thing to do. However, a layout generally only needs to be defined once and its rules don't need to be inherited. Using child and sibling selectors are very useful in this regard.

You can also see the maximum number of selectors in a rule. We've found that using more than four selectors is a good indicator of a problem with the implemented cascade. So keep your specificity as low as possible.

  • 2 column, left sidebar

    <body> <div class="site"> <header></header> <section> <article></article> </section> <aside></aside> <footer></footer> </div> </body>
    Less code:
    .site { margin: 0 auto; max-width: @max-width; } .site > header, .site > footer { clear:both; } .site > section { float: right; width: 66.66%; overflow: hidden; } .site > aside { float: left; width: 33.33% overflow: hidden; }

  • 2 columns, right sidebar

    <body> <div class="site"> <header></header> <section> <article></article> </section> <aside></aside> <footer></footer> </div> </body>
    Less code:
    .site { margin: 0 auto; max-width: @max-width; } .site > header, .site > footer { clear:both; } .site > section { float: left; width: 66.66%; overflow: hidden; } .site > aside { float: left; width: 33.33%; overflow: hidden; }

  • 3 columns, 2 sidebars

    <body> <div class="site"> <header></header> <section> <section> <article></article> </section> <aside></aside> </section> <aside></aside> <footer></footer> </div> </body>
    Less code:
    .site { margin: 0 auto; max-width: @max-width; } .site > header, .site > footer { clear:both; } .site > section { float: right; width: 75%; overflow: hidden; } .site > aside { float: left; width: 25%; overflow: hidden; } .site > section > section { float: left; width: 66.66%; overflow: hidden; } .site > section > section + aside { float: left; width: 33.33%; overflow: hidden; }
