Play 2.0

Server-side web application integration

Play 2.0 is the ideal server-side web framework for these front-end patterns, because it is designed for HTML5 web development with jQuery, and integrates LESS on the server-side.

Adding Front-End Patterns to a new Play 2.0 project

Here’s how you can make the front-end patterns available to a new Play project.

  1. Remove the default CSS stylesheet, scripts and images
  2. Copy JavaScript, font and image files
  3. Copy the LESS source files
  4. Prefix included LESS files’ names with an underscore
  5. Replace the main page template
  6. Replace the default index page

Remove the default CSS stylesheet, scripts and images

Delete the contents of the Play project’s public/ folder, which will be replaced by LESS stylesheets.

Copy JavaScript, font and image files

Copy the scripts, styles/vanilla/fonts and styles/vanilla/images folders to the Play project’s public folder.

Copy the LESS source files

Copy the LESS source files from styles/vanilla/less/*.less to a new app/assets/stylesheets folder. Play will use the LESS compiler to compile these files, and make the resulting CSS available to your view templates.

Prefix included LESS files’ names with an underscore

In app/assets/stylesheets, for each file except main.less, prefix the file name with an underscore and edit the file names in the @import statements in main.less. This stops Play trying to compile these files, which will fail, because they depend on each other. Instead, compiling main.less will result in the other files being compiled via the @import statements.

Replace the main page template

Replace the contents of app/views/main.scala.html with the following template. This template adds a subtitle parameter, includes the JavaScript files and replaces the body with a default template.

@(title: String, subtitle: String)(content: Html) @import play.api.Play.current <!DOCTYPE html> <html> <head> <title>@title</title> <link rel="stylesheet" media="screen" href="@routes.Assets.at("stylesheets/main.css")"> <link rel="shortcut icon" type="image/png" href="@routes.Assets.at("images/favicon.png")"> <script src="@routes.Assets.at("scripts/modernizr.js")" type="text/javascript" charset="utf-8"></script> <script src="@routes.Assets.at("scripts/jquery.js")" type="text/javascript" charset="utf-8"></script> <script src="@routes.Assets.at("scripts/jquery-ui.js")" type="text/javascript" charset="utf-8"></script> <script src="@routes.Assets.at("scripts/init.js")" type="text/javascript" charset="utf-8"></script> </head> <body> <header> <div class="logo">@play.api.Play.configuration.getString("application.name")</div> </header> <nav> <ul> <li class="active"><a href="/">Home</a></li> </ul> </nav> <article class="content"> <hgroup> <h1>@title</h1> <h5>@subtitle</h5> </hgroup> @content </article> <footer> <p>Copyright ©2011 Lunatech Labs BV</p> </footer> </body> </html>

Replace the default index page

Replace the contents of app/views/index.scala.html with the following template.

@(message: String) @main("Home", "Welcome to Front-End Patterns") { <p>@message</p> }
